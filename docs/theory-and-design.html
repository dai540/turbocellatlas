Guides

Why the system is split into a biological embedding layer and a compressed retrieval layer.

TurboCell Atlas treats semantic fidelity and retrieval efficiency as different jobs. A strong embedding model captures biology. TurboQuant makes candidate search lighter. Exact reranking protects the final ranking.

Design commitments

  • keep representation and retrieval separate
  • allow approximation only in candidate generation
  • treat metadata as part of the retrieval model
  • keep executed benchmarks as first-class artifacts

Architecture

tca.config validates runtime settings. tca.quantization owns TurboQuant encoding. tca.pipeline owns filtering, candidate generation, and reranking. tca.benchmarks keeps alternative backends isolated from the core.
