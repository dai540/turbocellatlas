What TurboCell Atlas can do, in plain language.

This page is the shortest explanation of where the package is already useful, what kind of biological questions it helps with, and where its current limits still are.

In plain language

TurboCell Atlas helps you start from one cell state that you care about and retrieve similar cells from a large atlas. The output is not just a ranked list. It is a small bundle of tables and figures that help you judge whether the result makes biological sense and whether compressed retrieval changed the answer.

What it can already do well

Find cells similar to a rare or coherent disease state

This is the strongest current use case. In the executed IPF myofibroblast centroid example, TurboQuant preserved the exact top-100 neighborhood while using substantially less candidate-layer memory than the original float32 embedding bank.

This makes the package useful when the target state is compact, the biology is coherent, and you want to compare compressed retrieval with exact retrieval.

Start from one interesting cell

You do not always need a curated centroid. In the executed single-cell example, one IPF myofibroblast cell was already enough to recover a coherent local neighborhood. This is helpful when you only trust a few cells or are working from a small pilot dataset.

Restrict retrieval to a biologically relevant cohort

Many biological questions are not really "search the whole atlas" questions. They are "search the atlas, but only inside IPF, only inside lung, or only inside one study" questions. TurboCell Atlas supports metadata filters so that your returned cells stay inside a meaningful biological context.

Compare retrieval methods and settings

The project includes exact search, HNSW, FAISS-PQ, and TurboQuant-based retrieval, together with executed benchmark artifacts. That makes it useful for teams who need to discuss whether a compressed method is changing the biological answer.

What it cannot yet do perfectly

TurboCell Atlas is still a research prototype. TurboQuant is not yet faster than exact search in this Python implementation, broad heterogeneous queries remain harder than compact rare states, and the strongest executed dataset is still the public SCimilarity tutorial dataset.

Good first use cases

  • Retrieve neighbors for a rare pathological fibroblast-like state.
  • Start from one representative cell and inspect its local neighborhood.
  • Compare whole-atlas retrieval with disease-restricted retrieval.
  • Generate a benchmark artifact bundle for discussion with collaborators.
