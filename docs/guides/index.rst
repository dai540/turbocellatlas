Guides
======

Guides sit between the first tutorial and the API reference.

They are especially useful for readers who are not starting from code. A guide should answer the kinds of questions that come up in meetings and collaborative analysis work: what the package is for, what needs to be prepared before a first run, and how to read the resulting tables and figures responsibly.
