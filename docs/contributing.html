Contributing to TurboCell Atlas

The project values reproducible benchmark claims, clear documentation, and changes that remain understandable to both computational and wet-lab collaborators.

  • Run pytest before opening a pull request.
  • Update docs if public behavior changed.
  • Explain any new data or model dependency.
  • Keep benchmark claims tied to generated artifacts.

For detailed notes, see the repository root CONTRIBUTING.md.
