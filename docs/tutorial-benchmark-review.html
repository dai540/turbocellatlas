Review order

input_data_dictionary.csv → pipeline_stages.csv → query_definitions.csv → candidate_rerank_comparison.csv → benchmark_summary.csv → top_hits.csv.

ArtifactWhat it answers
input_data_dictionary.csvWhat exactly was in the input dataframe.
pipeline_stages.csvWhat transformations were applied before retrieval.
query_definitions.csvWhat biological populations defined the queries.
candidate_rerank_comparison.csvHow candidate generation differed from the final reranked result.
benchmark_summary.csvWhat tradeoff each backend achieved.
top_hits.csvWhat cells were actually returned.
artifact_index.csvWhat else exists in the bundle and why.
Strong bundle

Inputs are named, process stages are explicit, candidate diagnostics are visible, and outputs are concrete tables and figures.

Weak bundle

Only one final score table exists and no one can tell what data, queries, or candidate behavior produced it.

This review pattern is the bridge between development output and stakeholder-facing evidence.
