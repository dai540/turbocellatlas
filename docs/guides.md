# Guides

Guides collect the conceptual and operational material that sits between the first tutorial and the API reference.

They are especially important for readers who are not starting from code. A guide should answer the kind of question that comes up in meetings and collaborative analysis work: what is this package for, what do we need before we can use it, how should we choose a query, and how should we read the output table once it comes back.

## Included guides

- [What TurboCell Atlas Can Do](what-turbocellatlas-can-do.md): the shortest plain-language summary of where the package is already useful
- [What You Need](what-you-need.md): the minimum ingredients you should prepare before running a search
- [Wet Lab Guide](wet-lab-guide.md): easiest explanation of inputs, outputs, and how to read the results
- [Theory and Design](theory-and-design.md): why the retrieval stack is split into embedding and compressed candidate layers
- [Benchmark Review](tutorial-benchmark-review.md): how to inspect the output bundle as a reviewer
- [Real Data Case Study](real-data-case-study.md): the full executed benchmark article

## What belongs in a guide

Guides answer questions like:

- what should the input metadata look like
- what can this package already do well, and where is it still weak
- what needs to be prepared before the first run
- when should a query be a centroid versus a single cell
- how should a retrieval table be interpreted biologically
- how should benchmark artifacts be reviewed before sharing them
