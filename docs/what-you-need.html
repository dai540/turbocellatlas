What you need before you can use TurboCell Atlas well.

This page explains the minimum ingredients for a successful first run: a reference atlas, a biologically meaningful query, and enough metadata to interpret the answer.

Minimum technical ingredients

You need three things: a reference atlas, a query, and a way to interpret the returned cells.

A reference atlas

This is the collection of cells you want to search against. In practice, TurboCell Atlas expects atlas embeddings plus metadata. At minimum, the metadata should contain cell_id and whatever fields you may want to filter on, such as Disease, tissue, study, sample, or celltype_name.

A query

The query is the cell state you want to search with. It can be one single cell, a centroid of several labeled cells, or a query vector saved in .npy. The most important practical question is not only how to compute a vector, but also what biological state that vector actually represents.

A way to interpret the results

TurboCell Atlas returns ranked cells, but ranks need biological context. You therefore need metadata fields that mean something biologically, a plan for checking whether the returned cells match expectation, and a way to compare filtered and unfiltered retrieval when cohort effects matter.

Minimum files for the easiest start

  • configs/wetlab_metadata_template.csv
  • notebooks/wet_lab_walkthrough.ipynb
  • artifacts/wetlab_examples/

Optional but strongly recommended

Meaningful metadata makes the package far more useful. Disease, tissue, study, sample, and cell type annotations are usually the most interpretable fields. It also helps to keep a benchmark mindset by comparing exact search, compressed search, and cohort-filtered search so you can tell whether the answer is stable.

Common misunderstandings

"I only have raw counts, not embeddings"

That is still workable, but TurboCell Atlas mainly expects embeddings at retrieval time. In the current examples, SCimilarity is used as the embedding layer.

"I only have one interesting cell"

That is fine. The single-cell tutorial shows that one cell can still be a valid starting point.

"I do not know which metadata fields matter"

Start with disease, tissue, study, sample, and cell type if you have them. Those are usually the most interpretable fields.
